Design Principles

  • Write your business logic once and reuse it across all instances
  • It should be easy to use
  • It should help reduce boilerplate
  • It should be composable
  • It should not require changing your code majorly
Review some of the known systems

Tried namespaced action, reducer and sagas

  • Very difficult to wire/use
  • Need to pass the same namespace in different places

Tried Kea, high level abstraction between React and Redux -

  • They have their own implementation of everything
  • Implement some of the saga effects

Our solution - develop a better system

  • It is as easy as reusing a component
  • It combines the component and its business logic in one bundle/package
  • Providing a namespace it's enough to produce a new instance
  • Call it Feature

ListFeatureComponent configuration

Implemented on top of the createFeature, which is resued accross Features. It:

  • takes a few paramaters - component, actions, etc.
  • will register reducer and sagas dynamically
  • will inject all state values defined in the reducer and action creators into the component as props
